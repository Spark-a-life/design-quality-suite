# Surface Reference -- Dashboard and Control Plane

Use this reference for operator dashboards, admin shells, internal tools, and
control-plane interfaces.

## Priorities

1. State clarity
2. Hierarchy and scan speed
3. Evidence of truth boundaries
4. Input and action safety

## Layout guidance

- Prefer structured asymmetry over generic equal-card grids.
- Keep key metrics and current-state signals near the top.
- Use monospace for IDs, counts, packet numbers, and timestamps.
- Let density rise only where it improves scan speed.

## Visual guidance

- Use one accent family and restrained lighting.
- Avoid theatrical gradients, neon glows, and decorative motion.
- Keep panels readable under operational use, not only in screenshots.

## Interaction guidance

- Buttons and nav items need clear hover, active, and focus states.
- Read-only vs simulated vs governed states should be explicit where relevant.
- If an action is not real yet, the UI must not imply live authority.
